About Merlin Yang

Hi! I’m Merlin. I build things that are practical and actually get used — from AI tools and web apps to game prototypes. My approach is simple: ship fast, learn quickly, and double down only on what adds real value. I care about clean systems, readable code, and outcomes that make sense outside of a demo.

Quick Snapshot

  • Currently: Master’s student in Computer Science at Northeastern, based in Oakland. Balancing school, side projects, and looking for my next opportunity.
  • Strengths: React + Node + Supabase for shipping quickly, PixiJS/Godot for games, Python when machine learning or data work actually moves the needle.
  • Philosophy: Build small, test fast, and don’t waste time on features nobody needs.
  • Big picture: I’d like to see tech push us toward a Star Trek–like future — knowledge open, tools accessible, and learning made playful.

How I Roll

  • Value first: If it doesn’t help users, it doesn’t ship.
  • Readable code: Clear structure, boring names, easy hand-off.
  • Experiment, then refine: Exploration is messy, but what goes live is measured and maintainable.
  • Security basics: No exposed secrets, predictable auth, least privilege.
  • Performance: Profile, then optimize what users actually notice.

Outside Code

When I’m not coding, I’m usually training at the gym (currently fixing years of skipping leg day), cooking, or taking photos. I’ve also got a soft spot for games that punish greedy movement — they keep me sharp.

Let’s Connect

I’m down to collab on web apps, AI side projects, or game prototypes. If you’re building something real and want a pragmatic engineer, let’s talk.
